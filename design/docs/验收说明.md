# 验收说明

## SQL 执行顺序

1. 使用 openGauss 管理账号执行 `design/sql/01_create_database.sql`。
2. 连接数据库 `DengjxMIS13`。
3. 依次执行 `02_schema.sql`、`03_views.sql`、`04_triggers.sql`、`05_procedures.sql`、`06_seed_data.sql`。
4. 检查基本表、视图、触发器、存储过程和初始化数据是否存在。

## 管理员闭环验证

1. 使用管理员账号登录。
2. 维护专业、班级、学生、教师、课程基础数据；专业需维护毕业所需学分，学生需维护入学时间。
3. 在课程管理中维护课程类型、开设年级和开设学期。
4. 创建开课安排，课程、班级、教师应通过下拉选择，上课时间应通过选择当日第几节课生成，不需要手动输入编号或具体时间；同一学年学期内教师、班级、教室时间冲突应被拒绝，课程容量不能超过教室容量。
5. 在选课记录管理中查看学生姓名、学号、课程、教师和状态，并验证全局选课开关；管理员新增有效选课记录时，只能选择该学生当前年级学期对应班级课程，不能绕过容量和课表冲突校验。
6. 在期末考试管理中维护课程考试时间；课程已有完成记录后，新增、修改或删除对应期末考试应被拒绝。
7. 在转专业审核中关闭全局转专业申请开关，学生转专业申请页面应显示“当前转专业申请未开放”提示且不展示提交表单；重新开启后学生可提交申请。
8. 审核通过转专业申请时选择目标专业下班级，当前学期学生个人信息、可选课程、我的选课、课表和班级同学仍按原班级专业展示，下一个学期起按新班级专业生效。
9. 在成绩管理中通过有效选课记录下拉录入或修改成绩。
10. 在统计查询中通过分类标签页查看地区学生数、课程平均分、学分汇总、教师任课、班级开课、学年成绩和课程排名；学生学年成绩和课程排名应通过下拉选择查询对象。

## 前端页面验证

1. 执行 `npm run build`，确认 TypeScript 和 Vite 构建通过。
2. 执行 `npm run dev`，打开 `/login`。
3. 检查登录页用户名、密码输入框可输入，“登录”按钮可点击。
4. 后端服务启动后，再使用管理员、教师、学生账号分别登录进入对应工作台。

## 教师闭环验证

1. 使用教师账号登录。
2. 在“我的任课”查看本人课程。
3. 在“我的任课”中点击“名单”，应自动进入对应课程选课名单并加载数据。
4. 在“我的任课”中点击“录成绩”，应自动进入对应课程成绩录入页并显示已录、未录人数。
5. 选择学生选课记录录入或修改成绩。
6. 在“我的任课”中点击“排名”，或在课程统计中选择课程，查看平均分和排名。

## 学生闭环验证

1. 使用学生账号登录。
2. 在学生中心确认当前年级和上/下学期显示正确。
3. 在“可选课程”查看当前年级学期开放的专业选修课并提交选课；管理员关闭全局选课开关后，页面不应展示可选课程，应显示“当前不在有效选课时间范围内！”；必修课不应出现在可选课程中。
4. 选课成功后应进入“我的选课”。
5. 在“我的选课”查看已选和已完成课程，选课功能开放时选修且未录成绩课程可退课，退课后该记录不再显示在我的选课页面，必修课不能退课；管理员关闭全局选课开关后，已选课程也不能退课。
6. 在“期末考试”只查看本人已选课程的期末考试，已完成或未选择课程不应出现在列表中。
7. 在“我的成绩”按年级学期查看成绩。
8. 在“我的学分”查看已修学分、毕业所需学分和剩余学分。
9. 在“我的班级”查看本班学生名单。
10. 在“转专业申请”提交目标专业和理由，并在管理员审核后看到审核结果；管理员关闭转专业申请开关时，页面应显示未开放提示且不展示提交表单。
11. 在“我的选课”中点击“排名”，或在“我的排名”选择本人已选课程查看课程排名；暂无成绩时应显示提示。

## 触发器验证

1. 给学生已选课程录入及格成绩。
2. 查询 `Dengjx_Students13.djx_TotalCredits13`，应自动增加对应课程学分。
3. 修改成绩为不及格或删除成绩后再次查询，总学分应自动重算。

## 存储过程验证

1. 调用 `Dengjx_GetStudentYearScores13`，检查指定学生和学年的课程、学期、成绩、学分、教师。
2. 调用 `Dengjx_GetCourseScoreRank13`，检查指定开课安排的成绩排名和课程平均分。

- `V_Dengjx_TeacherAssignments13`
- `V_Dengjx_ClassCourses13`
- `V_Dengjx_CourseAverage13`
