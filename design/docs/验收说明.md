# 验收说明

## SQL 执行顺序

1. 使用 openGauss 管理账号执行 `design/sql/01_create_database.sql`。
2. 连接数据库 `DengjxMIS13`。
3. 依次执行 `02_schema.sql`、`03_views.sql`、`04_triggers.sql`、`05_procedures.sql`、`06_seed_data.sql`。
4. 检查基本表、视图、触发器、存储过程和初始化数据是否存在。

## 管理员闭环验证

1. 使用管理员账号登录。
2. 维护地区、专业、班级、学生、教师、课程基础数据。
3. 创建开课安排，设置课程类型、容量和是否开放选课。
4. 在选课记录管理中新增、退选或恢复选课记录。
5. 在成绩管理中录入或修改成绩。
6. 在统计查询中查看地区学生数、课程平均分、学分汇总、教师任课、班级开课、学年成绩和课程排名。

## 前端页面验证

1. 执行 `npm run build`，确认 TypeScript 和 Vite 构建通过。
2. 执行 `npm run dev`，打开 `/login`。
3. 检查登录页用户名、密码输入框可输入，“登录”按钮可点击。
4. 后端服务启动后，再使用管理员、教师、学生账号分别登录进入对应工作台。

## 教师闭环验证

1. 使用教师账号登录。
2. 在“我的任课”查看本人课程。
3. 输入开课 ID 查看选课名单。
4. 选择学生选课记录录入或修改成绩。
5. 在课程统计中查看平均分和排名。

## 学生闭环验证

1. 使用学生账号登录。
2. 在“可选课程”查看开放选课课程并提交选课。
3. 在“我的选课”查看选课状态，未录成绩课程可退课。
4. 在“我的成绩”查看成绩。
5. 在“我的学分”查看触发器维护的已修学分。
6. 在“我的排名”输入开课 ID 查看课程排名。

## 触发器验证

1. 给学生已选课程录入及格成绩。
2. 查询 `Dengjx_Students13.djx_TotalCredits13`，应自动增加对应课程学分。
3. 修改成绩为不及格或删除成绩后再次查询，总学分应自动重算。

## 存储过程验证

1. 调用 `Dengjx_GetStudentYearScores13`，检查指定学生和学年的课程、学期、成绩、学分、教师。
2. 调用 `Dengjx_GetCourseScoreRank13`，检查指定开课安排的成绩排名和课程平均分。

## 视图验证

重点检查以下视图是否能返回预期数据：

- `V_Dengjx_StudentGrades13`
- `V_Dengjx_TeacherAssignments13`
- `V_Dengjx_ClassCourses13`
- `V_Dengjx_CourseAverage13`
- `V_Dengjx_RegionStudentCount13`
- `V_Dengjx_StudentCreditSummary13`
