# 接口清单

## Auth APIs

| 方法 | 路径 | 角色 | 用途 |
|---|---|---|---|
| POST | `/api/auth/login` | 公开 | 用户登录，返回 JWT、角色和显示名称 |

## 安全响应约定

- 未携带 JWT、JWT 无效或登录已过期时，受保护接口返回 HTTP 401。
- JWT 有效但角色无权访问目标接口时，返回 HTTP 403。
- 后端会对 401/403 安全拒绝请求输出 WARN 日志，包含状态码、请求方法、路径、远端地址和拒绝原因，不记录 JWT 内容。

## 错误响应约定

- 业务校验失败、参数不合法、权限不足、数据库唯一约束、外键约束和检查约束失败时，响应体 `message` 返回可直接展示给用户的具体原因。
- 未识别的系统级异常仍返回 `系统异常`，避免把 SQL、堆栈或内部实现细节暴露给前端。

### 学生 CSV 导入格式

管理员调用 `/api/admin/students/import` 时使用 `multipart/form-data` 上传字段名为 `file` 的 CSV 文件。CSV 第一行必须是表头：

前端入口位于管理员端“学生管理”页面的“CSV导入”弹窗。管理员可点击“选择CSV文件”从本机选择 `.csv` 文件，也可将 CSV 文件拖拽到上传区域；选择文件后点击“开始导入”提交。

```csv
sno,sname,gender,age,classId,regionId,admissionDate
```

字段说明：

| 字段 | 说明 |
|---|---|
| `sno` | 学号，不能为空，数据库中必须唯一 |
| `sname` | 姓名，不能为空 |
| `gender` | 性别，只能填写 `MALE` 或 `FEMALE` |
| `age` | 年龄，必须为 15 到 35 之间的整数 |
| `classId` | 所属班级编号，必须对应已存在班级 |
| `regionId` | 生源地区编号，必须对应已存在地区 |
| `admissionDate` | 入学时间，格式为 `yyyy-MM-dd`，不能晚于当前日期 |

导入时每行独立处理：合法行会写入学生表并创建账号，非法行会在响应的 `errors` 中返回具体行号和原因；如果账号创建失败，该行已插入的学生记录会同步清理，避免出现无账号学生。

合法学生导入成功后，系统会自动创建启用状态的学生账号：

| 项目 | 规则 |
|---|---|
| 用户名 | `s` + 学号，例如学号 `20269901` 对应账号 `s20269901` |
| 初始密码 | `123456` |
| 角色 | `STUDENT` |

测试 CSV 示例文件位于 `design/test-data/students-import-sample.csv`。
