# 管理员端查询修复说明

## 问题原因

管理员端查询不可用主要有两类原因：

- 部分自定义管理页只加载分页数据，没有查询输入框，也没有把 `keyword` 传到后端，例如选课记录管理、成绩管理。
- 部分后端列表接口虽然接收 `keyword`，但只查本表少数字段，未覆盖页面上实际显示的关联名称，例如课程名、班级名、教师名、专业名、教学楼名。

## 修复范围

- 选课记录管理：支持按学号、姓名、班级、课程、教师、学年、状态查询。
- 成绩管理：支持按课程编号、课程名称、课程类型、班级、教师、学年查询教学班。
- 通用 CRUD 管理页：扩展开课安排、专业课程计划、班级、教室、期末考试的关联字段查询。

## 验证

- 后端：`mvn test`
- 前端：`npm run build`
