# 高校教务系统设计说明

## 1. 项目目标

本项目按照 `设计要求.md` 实现一个高校教务系统，用于完成数据库课程设计，并作为全程使用 Codex 开发项目的学习实践。

系统采用前后端分离架构：

- 后端：Spring Boot 3、Java 17、MyBatis-Plus、Spring Security、JWT。
- 前端：Vue 3、Vite、TypeScript、Element Plus、Pinia、Vue Router、Axios。
- 数据库：openGauss。

系统需要覆盖数据库课程设计要求中的需求分析、概念结构、逻辑结构、物理结构、SQL 实现、视图、触发器、存储过程、业务系统开发和实验报告素材准备。

## 3. 核心业务模型

系统核心对象包括地区、专业、班级、学生、教师、课程、开课安排、选课记录、成绩和用户账号。

### 3.1 数据表设计

至少设计以下基础表：

| 表名 | 作用 |
|---|---|
| `Dengjx_Regions13` | 地区信息，用于统计生源地学生数。 |
| `Dengjx_Majors13` | 专业信息。 |
| `Dengjx_Classes13` | 班级信息，关联专业。 |
| `Dengjx_Students13` | 学生信息，关联班级和地区，保存已修学分总数。 |
| `Dengjx_Teachers13` | 教师信息。 |
| `Dengjx_Courses13` | 课程基础信息，保存课程编号、名称、学时、考核方式、学分。 |
| `Dengjx_TeachingAssignments13` | 开课安排，关联课程、班级、教师、学年、学期、课程类型、容量和选课状态。 |
| `Dengjx_Enrollments13` | 学生选课记录，关联学生和开课安排。 |
| `Dengjx_Grades13` | 成绩信息，关联选课记录。 |
| `Dengjx_Users13` | 登录账号，支持管理员、教师、学生三类角色。 |

### 3.2 关键关系

- 一个专业可以包含多个班级。
- 一个班级可以包含多个学生。
- 一个学生属于一个班级和一个生源地区。
- 一门课程可以在不同学年、学期、班级下多次开设。
- 一个开课安排由一个教师授课，面向一个班级或选课范围。
- 一个学生可以选择多个可选课程。
- 一个选课记录最多对应一条成绩记录。
- 未选课的学生不能录入成绩。

## 4. 课程与选课设计

开课安排表 `Dengjx_TeachingAssignments13` 增加课程类型字段：

- `REQUIRED`：班级必修课，由管理员按班级发布。
- `ELECTIVE`：可选课程，学生可以主动选课。

选课记录表 `Dengjx_Enrollments13` 保存学生对开课安排的选择状态：

- `SELECTED`：已选。
- `DROPPED`：已退。
- `COMPLETED`：已完成。

核心规则：

- 学生只能选择开放选课的课程。
- 同一学生不能重复选择同一开课安排。
- 课程人数不能超过容量。
- 已录入成绩的课程不允许学生退课。
- 成绩必须基于有效选课记录录入。

## 5. 用户角色与功能矩阵

系统包含三类角色：管理员、教师、学生。

| 模块 | 管理员 ADMIN | 教师 TEACHER | 学生 STUDENT |
|---|---|---|---|
| 账号与登录 | 登录、退出、修改密码、管理用户账号 | 登录、退出、修改密码、查看本人信息 | 登录、退出、修改密码、查看本人信息 |
| 地区管理 | 地区增删改查、地区学生数统计 | 无 | 无 |
| 专业管理 | 专业增删改查 | 查询相关专业 | 查看本人专业 |
| 班级管理 | 班级增删改查 | 查询任课班级 | 查看本人班级 |
| 学生管理 | 学生增删改查、创建学生账号 | 查询任课课程下的学生 | 查看本人信息 |
| 教师管理 | 教师增删改查、创建教师账号 | 查看本人信息 | 查看任课教师信息 |
| 课程管理 | 课程增删改查 | 查询本人相关课程 | 查询可选课程和已选课程 |
| 开课安排 | 创建班级必修课、创建可选课程、设置教师、学年、学期、容量和选课状态 | 查看本人任课安排 | 查看本人班级必修课、查看可选课程 |
| 学生选课 | 查看、新增、退选、恢复选课记录，处理异常选课 | 查看本人课程的选课名单 | 选课、退课、查看已选课程 |
| 成绩管理 | 录入、修改、删除所有成绩 | 录入、修改本人任课课程中已选课学生的成绩 | 查看本人成绩 |
| 学年成绩统计 | 查询任意学生 | 查询本人任课学生 | 查询本人 |
| 成绩排名 | 按课程、班级、学年查询 | 查询本人任课课程排名 | 查看本人在课程或班级中的排名 |
| 课程平均分 | 查询全部课程 | 查询本人任课课程 | 查看本人所学课程平均分 |
| 课程与学分统计 | 查询任意学生课程和学分 | 查询本人任课学生课程完成情况 | 查看本人课程、已修学分 |
| 教师任课查询 | 查询全部教师任课 | 查询本人任课 | 查看本人课程对应教师 |
| 班级课程查询 | 查询全部班级开课 | 查询本人任课班级开课 | 查询本人班级开课 |
| 选课名单查询 | 查询任意开课安排名单 | 查询本人任课课程名单 | 查看自己的选课记录 |
| 系统仪表盘 | 学生数、教师数、课程数、开课数、选课数、成绩数 | 任课数、选课学生数、已录成绩数 | 已选课程数、已修学分、平均分 |

三类用户闭环：

- 管理员：维护基础数据 -> 发布课程和开课安排 -> 管理选课 -> 管理成绩 -> 查看统计。
- 教师：查看任课 -> 查看选课名单 -> 录入成绩 -> 查看课程统计。
- 学生：查看可选课程 -> 选课或退课 -> 查看已选课程 -> 查询成绩和学分。

## 6. 数据库对象要求

### 6.1 约束

数据库表需要包含以下完整性约束：

- 主键约束。
- 外键约束。
- 唯一约束，例如学号、教师编号、课程编号、用户名。
- 检查约束，例如性别、年龄、成绩范围、学分、学时、课程类型、选课状态。
- 非空约束。

删除策略：

- 不默认使用危险级联删除。
- 对已有成绩、选课、开课安排等关联数据的核心记录，删除时应由外键约束阻止，并在后端返回明确错误。

### 6.2 视图

至少建立以下视图：

- 学生成绩查询视图。
- 教师任课查询视图。
- 班级课程开设查询视图。
- 每门课程平均成绩统计视图。
- 地区学生数统计视图。
- 学生课程与学分统计视图。

### 6.3 触发器

至少建立两个触发器：

- 成绩新增或修改后，自动重算学生已修学分总数。
- 成绩删除后，自动重算学生已修学分总数。

已修学分统计规则：

- 只统计有效成绩。
- 成绩及格线按 60 分处理。
- 只统计学生已经选课并完成成绩录入的课程学分。

### 6.4 存储过程

至少建立两个存储过程：

- 学生按学年统计成绩。
- 按课程统计平均成绩和成绩排名。

## 7. 后端设计

后端使用 Spring Boot 分层结构：

- `controller`：处理 HTTP 请求、参数校验、角色限制和响应包装。
- `service`：处理业务规则。
- `mapper`：处理 MyBatis-Plus 数据访问。
- `entity`：数据库实体。
- `dto`：请求和响应对象。
- `security`：JWT、登录认证、角色权限。
- `common`：统一响应、分页、异常处理。

### 7.1 模块划分

| 模块 | 职责 |
|---|---|
| `auth` | 登录、退出、当前用户信息、密码修改。 |
| `region` | 地区维护和地区学生数统计。 |
| `major` | 专业维护。 |
| `class` | 班级维护。 |
| `student` | 学生维护、学生个人信息和学生学分查询。 |
| `teacher` | 教师维护、教师任课查询。 |
| `course` | 课程基础信息维护。 |
| `assignment` | 开课安排维护。 |
| `enrollment` | 学生选课、退课、选课名单查询。 |
| `grade` | 成绩录入、修改、删除、查询、排名。 |
| `statistics` | 学年成绩、课程平均分、地区学生数、学分统计。 |

### 7.2 权限规则

- 管理员拥有基础数据、开课、选课、成绩和统计的完整管理权限。
- 教师只能查看自己的任课安排、选课名单，并录入或修改本人任课课程的成绩。
- 学生只能查看和操作自己的选课、成绩、课程、学分数据。
- 前端菜单权限只用于体验控制，后端接口权限才是安全边界。

### 7.3 业务校验

- 成绩范围为 0 到 100。
- 同一学生同一开课安排只能有一条有效选课记录。
- 同一有效选课记录只能有一条成绩。
- 教师录入成绩时，必须校验该开课安排属于当前教师。
- 学生查询数据时，必须校验当前账号关联的学生编号。
- 管理员删除基础数据时，如果存在关联记录，应提示存在关联数据，不进行删除。

## 10. 开发阶段

开发按以下阶段推进：

1. 数据库阶段：建库、建表、约束、索引、视图、触发器、存储过程、初始化数据。
2. 后端基础阶段：初始化 Spring Boot、连接 openGauss、统一响应、异常处理、JWT 权限。
3. 后端业务阶段：基础资料、开课安排、选课、成绩、统计接口。
4. 前端基础阶段：初始化 Vue、登录、布局、路由、权限菜单、Axios 封装。
5. 前端业务阶段：管理员、教师、学生三端页面。
6. 联调与验收阶段：接口联调、SQL 验证、准备实验报告截图素材和使用说明。

## 11. 验收标准

系统完成后应满足以下标准：

- openGauss 数据库脚本可执行。
- 数据库表数量不少于 6 张，且存在明确主外键关联。
- 表名、字段名、数据库名符合课程个性化命名要求。
- 至少包含 2 个触发器和 2 个存储过程。
- 至少包含满足查询和统计需求的视图。
- 管理员、教师、学生三类用户均能登录并完成各自业务闭环。
- 学生可以完成选课、查看已选课程、查看成绩和学分统计。
- 教师可以查看任课、查看选课名单、录入成绩和查看统计。
- 管理员可以维护基础数据、发布开课、管理选课和成绩、查看统计。
- 前后端可以完成主要业务联调。
- 项目文档和 SQL 脚本结构清晰，可用于实验报告整理。
